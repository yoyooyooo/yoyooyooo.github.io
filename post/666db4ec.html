Rxjs 安装

npm 安装或者直接引用在线版

npm install rxjs
https://unpkg.com/rxjs@6.1.0/index.js

Rxjs 是观察者模式和迭代器模式的组合。它 2 个重要的概念：Observable 和 Observer，即“可被观察的对象”和“观察者”，连接 2 者的桥梁的就是 Observable 对象的 subscribe 方法：可简单表示为 Observable$.subscribe(observer)。
Rxjs 中的数据流就是 Observable 对象，一般参数结尾加个$代表 Observable 对象
RXjs 实现的是“推”的迭代器模式，发布者会主动将数据推给观察者，而不需要主动从 Observable 中“拉”数据，只要用 subscribe 连接两者后，自然后能接收到数据推送。
observer 是普通对象，必须包含有 next 属性，属性值是个函数，用于接收被“推”过来的数据。

观察者模式带来的好处是双方都可以专心做一件事，而且可以任意组合，复杂的问题被分解成 3 个小问题：

  • 发布者产生“事件”，即 Observable 对象的工作
  • 观察者响应“事件”，即 observer 的工作
  • 什么样发布者关联什么样的观察者，也就是何时调用 subscribe

Hot Observable 和 Cold Observable

假如有这样一个场景，一个 Observable 对象有 2 个 observer 来订阅，而且不是同时订阅的，第一个 observer 订阅 N 秒后第二个 observer 才订阅，而且 N 秒后 Observable 已经吐出了一些数据，那么问题来了，后订阅的 observer 是不是应该收到“错过”的数据呢？

Rxjs 其实已经考虑到这 2 种场景了，后订阅的应该收到“错过”的数据的被称为 Cold Observable，不应该的被称为 Hot Observable。

Cold Observable 可以理解为每一次 subscribe 都产生一个数据“生产者”。
Hot Observable 可以理解为只是让 observer 连接数据“生产者”，只是数据的搬运工。

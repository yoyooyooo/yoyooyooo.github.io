umi插件笔记

umi 的核心是插件系统，可以写自己的插件针对功能的需要可以去使用修改代码打包配置，修改启动代码，约定目录结构，修改 HTML 等更丰富接口，具体官方文档写的挺详细的，这里记录下实际使用。

插件使用

插件可以是一个 npm 包，也可以是路径直接引向一个 JS 的模块。用户通过配置 plugins 来使用插件。

// .umirc.js
export default {
  plugins: [[join(__dirname, './plugins/test.js'), { a: 1 }]]
};

插件文件默认导出一个模块,第一个参数是 umi 提供的一系列 api，第二个参数是引入的时候传的 options

export default (api, opts) => {};

path

api.path能获取到一系列的路径

最常用的操作，配置改编重新生成缓存文件

api.onOptionChange(newOpts => {
  opts = newOpts;
  api.rebuildTmpFiles();
});

registerMethod

api.registerMethod('addCount', {
  type: api.API_TYPE.ADD
});
api.addCount(() => {
  return [1, 2, 3];
});
api.addCount(() => {
  return 4;
});
const result = api.applyPlugins('addYooo');
console.log(result); // [ 1, 2, 3, 4 ]
api.registerMethod('modifyCount', {
  type: api.API_TYPE.MODIFY
});
api.modifyCount(opt => {
  console.log('opt：', opt);
  return { a: 2 };
});
//如果是 api.applyPlugins('modifyCount', { a: 1 } ) ，上面执行的opt是undefined
const result = api.applyPlugins('modifyCount', { initialValue: { a: 1 } });
console.log(result);
// opt：{ a: 1 }
// { a: 2 }

applyPlugins 可以接受第二个参数，作为执行注册方法的函数参数的接收值，必须 { initialValue }才能被接收到

modifyRoutes

// /plugins/test.js
function patchRoutes(routes) {
  routes.forEach(route => {
    if (route.routes) {
      patchRoutes(route.routes);
    } else {
      if (route.path === '/b') {
        route.Routes = ['./routes/PrivateRoute.js', './routes/PrivateRoute2.js'];
      }
    }
  });
}

export default (api, opts) => {
  api.modifyRoutes(routes => {
    patchRoutes(routes);
    return routes;
  });
};

上面例子会让路径/b的页面加载./routes/PrivateRoute.js和./routes/PrivateRoute2.js2 个 layout 组件，

const route = {
  path: '/xxx',
  component: '/path/to/component',
  Routes: ['/permissionControl.js']
};

利用modifyroutes其实可以做很多关于目录约定的事，比如满足某些条件的时候加载约定 js 的组件：

route.Routes = [path.dirname(route.component) + '/_mylayout.js'];

_mylayout.js还能顺便注入点 props：

import React from 'react';

export default props => {
  const children = React.Children.map(props.children, child => {
    return React.cloneElement(child, { test: 'test' });
  });
  return (
    <div>
      <h1>PrivateRoute</h1>
      {children}
    </div>
  );
};

更深一步，可以结合 umi 自带的约定式路由实现更深入的约定。比如一个页面可以分为Head.js、Content.js、Footer.js等等文件，通过插件将它们组合起来，当然这样这几个组件就没有数据交流了，可以在第一个组件开始通过 React.cloneElement 传入把 props 传给 children，redux 也可以在第一个组件 connect，后面的 props umi 会自动传下去。不过这样的话这几个组件之间交流起来就麻烦了，，只能通过 React.cloneElement 传给下一层，所以如果要这样约定目录，最好是分成几块不相干的布局组件，不然会迎引来不必要的麻烦。

route.Routes = [
  dirname(route.component) + '/Head.js',
  dirname(route.component) + '/Content.js',
  dirname(route.component) + '/Footer.js'
];

onPatchRoute 也能做到和 modifyroutes 类似的事，后面再研究有什么区别。 感觉 onPatchRoute 是遍历 modifyroutes 的时候顺便暴露出来的单层路由

从 umi-plugin-react 中关于 title 部分的插件来看，一般是在 modifyroutes 里修改 routes，在 onPatchRoute 里向 Routes 中添加组件路径使得可以给路由添加一层封装。

api.onPatchRoute(({ route }) => {
  if (!route.routes || !route.routes.length) {
    if (route.path === '/b') {
      route.Routes = [
        path.dirname(route.component) + '/Header.js',
        path.dirname(route.component) + '/Content.js',
        path.dirname(route.component) + '/Footer.js'
      ];
    }
  }
});

addRendererWrapperWithModule

接收一个路径在挂载 前执行一个 Module，支持异步。就是在入口文件.umi/umi.js里引入指定模块，在挂载前先执行一次。（绝对路径，相对路径的话会以.umi/umi.js为基准）。.umi/umi.js里大概会生成以下代码：

const moduleBeforeRendererPromises = [];
if (typeof moduleBeforeRenderer0 === 'function') {
  const promiseOfmoduleBeforeRenderer0 = moduleBeforeRenderer0();
  if (promiseOfmoduleBeforeRenderer0 && promiseOfmoduleBeforeRenderer0.then) {
    moduleBeforeRendererPromises.push(promiseOfmoduleBeforeRenderer0);
  }
}

Promise.all(moduleBeforeRendererPromises)
  .then(() => {
    render();
  })
  .catch(err => {
    if (process.env.NODE_ENV === 'development') {
      console.error(err);
    }
  });

// render()

modifyEntryRender

修改.umi/umi.js里的 render 部分的代码，接受个函数参数，参数为原来的代码，可以进行替换或新增

api.modifyEntryRender(memo => {
  return `${memo}
    console.log(111111111)
`;
});

// 编译后的umi.js
let oldRender = () => {
  const rootContainer = window.g_plugins.apply('rootContainer', {
    initialValue: React.createElement(require('./router').default)
  });
  ReactDOM.render(rootContainer, document.getElementById('root'));
  console.log(111111111);
};

writeTmpFile

快捷得在.umi文件夹里生成文件（官方文档未指出该 api），一般配合onGenerateFiles使用：

api.onGenerateFiles(() => {
  api.writeTmpFile('aaaa.js', '1212');
});
